# Lick of the Day

Practice and pick up a new lick every day. AI-generated musical licks across jazz, R&B, funk, blues, and bossa nova, with interactive sheet music and a playable piano.

## Features

- **Daily lick**: One shared lick for everyone, pre-generated by a cron worker and refreshed every 24 hours
- **New lick on demand**: Generate random licks by genre (jazz, R&B, funk, blues, bossa nova) and length (2/4/6/8 bars). Generation streams over SSE so the sheet music paints as soon as it arrives, before the full note data finishes
- **Sheet music**: Rendered via abcjs with chord symbols
- **Interactive piano**: Click or use your computer keyboard (A-K = white keys, W/E/T/Y/U = sharps)
- **Listen mode**: Hear the lick played back with adjustable tempo
- **Practice mode**: Play notes in sequence, get instant feedback on correctness

## Tech Stack

- React + TypeScript + Vite
- abcjs (sheet music rendering)
- Tone.js (audio playback)
- Tailwind CSS
- Claude API (lick generation), streamed via Server-Sent Events

## Architecture

The app ships against two runtimes that are kept in lockstep:

- **Cloudflare Pages** (primary, live target): the page functions in `functions/api/*` run on the Workers runtime and read/write the daily lick from a KV namespace (`LICK_STORE`). Shared logic lives in `functions/_shared/*`.
- **Vercel** (parallel): the Node serverless handlers in `api/*` mirror the same behavior using the Anthropic SDK and import shared logic from `src/utils/*`.

Cloudflare and Vercel cannot import from each other's directories, so a few modules (prompt, parse, SSE helpers) are intentionally duplicated between `functions/_shared/*` and `src/utils/*`. Keep both copies in sync when changing shared logic.

The **daily lick** is pre-generated into KV by a separate scheduled worker (`workers/daily-lick-cron`, runs at midnight UTC), so `/api/daily` is almost always an instant cache hit. The interactive wait happens only on the cold path of `/api/random`, which is why that endpoint streams.

## Deployment

### Cloudflare Pages (primary)

Connect the repo to Cloudflare Pages with build output `dist` (see `wrangler.toml`). Bind the `LICK_STORE` KV namespace and set `ANTHROPIC_API_KEY` as an environment variable.

Deploy or redeploy the daily cron worker before deploying Pages. It owns the
strongly consistent coordinator used by stale-refresh and rate-limit admission:

```bash
cd workers/daily-lick-cron
npx wrangler deploy
```

### Vercel (parallel)

Set `ANTHROPIC_API_KEY` in environment variables, then:

```bash
vercel
```
